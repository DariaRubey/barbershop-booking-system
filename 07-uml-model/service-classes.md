Справочник «Роли сотрудника»  
**Назначение класса:** Испо Сущность предназначения для хранения записей о том, какие роли есть у конкретных сотрудников.льзуется при составлении расписания, а также для определения свободных мастеров при бронировании слота  
Один сотрудник может иметь одну или несколько ролей

Справочник «Услуги»  
**Назначение класса:**Сущность предназначена для хранения перечня услуг, которые оказывает барбершоп. Используется при выборе услуг клиентом, а также в справочнике «Услуги мастера» для определения того, какие услуги оказывает конкретный мастер.  
Один экземпляр сущности **«Услуги»** соответствует одной услуге барбершопа 

**Атрибуты**

| Ключ | Наименование атрибута | Мнемоника | Описание атрибута | Тип данных | Обязательность | Комментарий | значение по умолчанию |
| :---- | :---- | :---- | :---- | :---- | :---- | :---- | :---- |
| РК | Идентификатор  услуги | service\_id | Уникальный системный идентификатор услуги | integer  | да |  | Сгенерированный ID |
|  | Название услуги | service\_name | Название услуги | varchar (100) | да | Название услуги, отображаемое клиенту при выборе (например, «Мужская стрижка», «Моделирование бороды»). | Не задано |
|  | Статус услуги  | is\_active  | Текущее состояние услуги  | boolean  | да | Если true, услуга доступна для выбора. Если false, услуга скрыта/недоступна для новых записей | true |

Справочник «Услуги мастера»  
**Назначение класса:** Сущность которая показывает, какие услуги может выполнять конкретный мастер Сущность используется при бронировании слота: система должна показать клиенту только те услуги, которые оказывает выбранный мастер, или только тех мастеров, которые оказывают выбранную услугу   
Один экземпляр сущности **«Услуги мастера»** соответствует одной связи между сотрудником-мастером и услугой
