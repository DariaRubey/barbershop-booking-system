**Критерии приемки для US-BAR-01**   
Как мастер, я хочу просматривать свое расписание и записи клиентов, чтобы понимать свою загрузку 

1. Просмотр расписания на выбранную дату

**Given** мастер авторизован в системе  
**And** у мастера составлено расписание на выбранную дату  
**When** мастер открывает расписание  
**And** выбирает дату  
**Then** система должна показать расписание мастера на выбранную дату  
**And** в расписании должны отображаться свободные и занятые записями интервалы мастера

2. Просмотр деталей записи клиента

**Given** мастер авторизован в системе  
**And** в расписании есть активная запись клиента  
**When** мастер выбирает конкретную запись  
**Then** система должна открыть карточку записи  
**And** в карточке должны отображаться дата, время, услуга, клиент и статус записи.  
**Тестовые сценарии для US-BAR–02**  
**Как мастер, я хочу просматривать отзывы клиентов по своим услугам, чтобы понимать оценку своей работы и учитывать предложения клиентов** 

1\. Успешный сценарий. Мастер просматривает отзывы по своим услугам 

*Предусловие:* В системе есть выполненные записи клиентов к этому мастеру.  
После оказания услуг клиенты оставили оценки и текстовые отзывы.  

*Действие:* Мастер открывает раздел “Отзывы”.  
Система загружает отзывы, связанные с услугами этого мастера.  
Мастер просматривает список отзывов.  
Мастер открывает один из отзывов

*Ожидаемый результат:* Система показывает мастеру отзывы по его услугам.  
В отзыве отображаются оценка, текст отзыва, дата услуги и название услуги

2\. Неуспешный сценарий. Мастер видит отзывы других мастеров

*Предусловие:* В системе есть минимум два мастера  
У обоих мастеров есть отзывы клиентов

*Действие:* Мастер №1 открывает раздел “Отзывы”  
Система загружает отзывы, связанные с услугами Мастера №2  
Система загружает отзывы для отображения

*Ожидаемый результат:* Мастер №1 не видит своих отзывов

3\. Краевой сценарий. У мастера нет отзывов 

*Предусловие:*  В системе есть выполненные записи клиентов к этому мастеру  
В системе нет отзывов на этого мастера

*Действие:* Мастер открывает раздел “Отзывы”.  
Система загружает отзывы, связанные с услугами этого мастера

*Ожидаемый результат:* Система показывает сообщение, что отзывов пока нет.
