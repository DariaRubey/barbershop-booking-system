**Критерии приемки для US-CLT–01.**  
Как клиент, я хочу записаться на услугу онлайн, чтобы иметь возможность самостоятельно видеть свободные слоты

1. Успешная онлайн-запись на доступный слот  
   **Given** форма онлайн записи открыта  
   **and** расписание мастеров составлено и отражено в системе  
   **and** услуги мастеров отражены в системе  
   **when** клиент выбирает доступный слот (время, мастер, услуга)  
   and вводит свой номер телефона  
   and подтверждает запись   
   **then** система должна создать карточку записи  
   **and** карточка записи должна содержать клиента, номер телефона, услугу, мастера, дату, время и цену  
   **and** выбранный слот должен стать недоступным для других клиентов  
     
2. Отображение только доступных слотов  
   **Given** форма онлайн-записи открыта  
   **And** клиент выбрал услугу и мастера  
   **And** в системе есть расписание выбранного мастера  
   **When** клиент открывает список доступного времени  
   **Then** система должна показать только свободные слоты выбранного мастера  
   **And** система не должна показывать занятые или недоступные слоты.  
     
3. Отображение цены до подтверждения записи  
   **Given** форма онлайн-записи открыта  
   **And** клиент выбрал услугу  
   **And** выбранная услуга имеет цену в системе  
   **When** клиент переходит к подтверждению записи  
   **Then** система должна показать клиенту цену выбранной услуги до подтверждения записи.  
4. Нельзя создать запись без номера телефона

**Given** форма онлайн-записи открыта  
**And** клиент выбрал услугу, мастера и доступный слот  
**When** клиент не вводит номер телефона  
**And** пытается подтвердить запись  
**Then** система не должна создавать карточку записи  
**And** система должна показать сообщение о необходимости указать номер телефона.

5. Нельзя записаться на уже занятый слот

**Given** форма онлайн-записи открыта  
**And** клиент выбрал услугу, мастера и слот  
**And** выбранный слот уже был занят другим клиентом до момента подтверждения  
**When** клиент подтверждает запись  
**Then** система не должна создавать новую запись на этот слот  
**And** система должна показать сообщение, что слот больше недоступен  
**And** система должна предложить выбрать другой свободный слот.

**Тестовые сценарии для US-CLT–02**  
**Как клиент, я хочу получить напоминание о записи на услугу, чтобы не забыть посетить ее**

1\. Успешный сценарий. Напоминание о записи отправлено клиенту

*Предусловие:* В системе есть активная запись клиента более чем через 24 часа  
Менеджер настроил расписание напоминаний  
Наступает время отправки напоминания — за 24 часа до записи 

*Действие:* Система проверяет наличие активных будущих записей, находит нужную  
формирует текст напоминания  
отправляет напоминание клиенту  
Клиент получает сообщение с информацией о записи

*Ожидаемый результат:* Клиент получает напоминание,  
в сообщении указаны дата, время, услуга, мастер и адрес барбершопа.

2\. Неуспешный сценарий. Телефон клиента некорректный, напоминание не отправлено

*Предусловие:* В системе есть активная запись клиента более чем через 24 часа  
Менеджер настроил расписание напоминаний  
Наступает время отправки напоминания — за 24 часа до записи  
Номер телефона клиента указан в некорректном формате 

*Действие:* Система находит активную будущую запись клиента  
проверяет номер телефона клиента  
обнаруживает некорректный формат номера

*Ожидаемый результат:* Система не отправляет SMS 

3\. Краевой сценарий. Напоминание не отправляется по отмененной записи 

*Предусловие:*  В системе есть активная запись клиента более чем через 24 часа  
Менеджер настроил расписание напоминаний  
До времени отправки напоминания менеджер отменил запись 

*Действие:* Система проверяет наличие активных будущих записей, находит нужную  
Система видит, что статус записи \- “Отменена”.  
Система не формирует напоминание

*Ожидаемый результат:* Отмененная запись не участвует в автоматической рассылке напоминаний
