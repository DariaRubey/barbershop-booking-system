**Критерии приемки для US-MNG–01.**  
Как менеджер, я хочу составлять график мастеров в системе, чтобы клиенты могли самостоятельно записаться на услугу

1. **Given** системе есть хотя бы 1 пользователь с ролью мастер  
   **and** известны рабочее расписание мастера  
   **and** известны услуги мастера, которые он может оказывать,  
   **when** менеджер вносит расписание мастера в систему,  
   **then** система должна создать доступные слоты для онлайн записи клиентам в рамках указанного расписания.  
     
2. **Given** расписание мастера сохранено в системе,  
   **when** менеджер изменяет рабочее время мастера и сохраняет изменения,  
   **then** система должна обновить доступные слоты для онлайн-записи в соответствии с новым расписанием.  
     
3. **Given**  в системе есть активная запись клиента на слот мастера  
   **and** время оказания услуги еще не наступило  
   **and** клиент попросил перенести запись,  
   **when** менеджер выбирает новый доступный слот и сохраняет перенос записи,  
   **then** система должна занять новый слот, освободить предыдущий слот и обновить данные записи клиента.  
     
4. **Given**  в системе есть активная запись клиента на слот мастера  
   **and** время оказания услуги еще не наступило  
   **and** клиент попросил отменить запись,  
   **when** менеджер отменяет запись в системе,  
   **then** система должна изменить статус записи на «Отменена» и сделать ранее занятый слот доступным для онлайн-записи.  
   

**Тестовые сценарии для US-MNG–02**  
**Как менеджер, я хочу фиксировать выполнение услуги и принятие оплаты в системе, чтобы не вести учет оплат вручную отдельно**

1\. Успешный сценарий. Фиксация оплаченной услуги в системе

*Предусловие:* В системе есть активная запись клиента  
услуга уже оказана  
стоимость услуги указана в системе  
оплата еще не зафиксирована.

*Действие:* Менеджер открывает карточку записи  
отмечает услугу выполненной  
выбирает способ оплаты  
принимает оплату у клиента  
фиксирует оплату в системе.  
Система сохраняет данные оплаты.

*Ожидаемый результат:* В карточке записи отражен статус оплаты “оплачено”,  
сумма оплаты,   
статус услуги “выполнена”.

2\. Неуспешный сценарий. Фиксация оплаты с неполными данными

*Предусловие:* В системе есть активная запись клиента,   
услуга уже оказана,  
стоимость услуги указана в системе,   
оплата еще не зафиксирована.

*Действие:* Менеджер открывает карточку записи,   
отмечает услугу выполненной,   
не выбирает способ оплаты,  
сохраняет оплату.   
Система проверяет обязательные поля.

*Ожидаемый результат:* Система не сохраняет оплату,   
система показывает сообщение о необходимости выбрать способ оплаты 

3\. Краевой сценарий. Повторная фиксация оплаты по уже оплаченной записи 

*Предусловие:* В системе есть запись клиента со статусом “Выполнена”  
оплата по записи уже зафиксирована,  
Статус оплаты “Оплачено”.

*Действие:* Менеджер открывает карточку записи,   
пытается повторно нажать “Зафиксировать оплату”.  
Система проверяет текущий статус оплаты.

*Ожидаемый результат:* Система не создает повторную оплату по этой же записи,   
система показывает сообщение, что оплата уже зафиксирована.  
Сумма оплаты не дублируется.
