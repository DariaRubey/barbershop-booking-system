**User story**   
Как менеджер, я хочу вносить в систему актуальное расписание мастеров, чтобы клиенты могли записываться на услуги онлайн.

| **Описание** | Менеджер ведет актуальное расписание мастеров в системе. Менеджер создает новое расписание или вносит изменения в уже имеющееся расписание. Система сохраняет эти данные для дальнейшего использования при онлайн-записи и просмотре расписания мастером. |
| **Триггер** | Менеджеру нужно создать новое расписание (появился новый мастер) или изменить существующее (мастер хочет изменить свое рабочее время) |
| **Предварительные условия** | В системе есть справочник мастеров и справочник услуг В системе есть данные хотя бы об одном мастере Менеджер авторизован в системе У менеджера есть права доступа на ведение расписания |
| **Выходные условия** | Минимальные гарантии: Неподтвержденные изменения не сохраняются, старое расписание остается как былоГарантии успеха: Расписание работы мастера сохранено в системе. Обновленные данные доступны для использования в системе онлайн-записи и для просмотра мастером. |
| **Нормальное направление развития варианта использования** | **Вести расписание мастеров** Менеджер открывает раздел “расписание мастеров” Менеджер выбирает мастера и период расписания Система показывает действующее расписание Менеджер вносит рабочие периоды мастера (дни, время работы) Система подтверждает  корректность внесенного расписания Менеджер подтверждает сохранение расписания Система сохраняет расписание работы СИстема сообщает об успешном сохранении данных |
| **Альтернативное направление развития варианта использования** | **2А. Данные введены некорректно** Система сообщает об ошибке при вводе  Менеджер повторно вводит данные в корректном формате Система подтверждает соответствие обновленных данных правилам ведения расписания.  UC продолжается с шага 6 основного сценария.   |
