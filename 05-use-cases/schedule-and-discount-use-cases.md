| **Основное действующее лицо** | Клиент |
| **Роль действующего лица** | Человек, который хочет записаться на услугу онлайн |
| **Потребность** | Иметь личный кабинет для хранения информации о записях |
| **Бизнес-цель** | Использовать учетную запись клиента при создании записи.  |

| **Тип связи** | Extend. Авторизация не обязательна. По условию задачи записаться может и зарегистрированный, и незарегистрированный посетитель сайта. Основной UC должен успешно выполняться без авторизации.  |
| **Описание** | Клиент входит в существующую учетную запись или регистрируется во время записи, чтобы система использовала данные профиля при бронировании услуги. Если клиент не хочет входить или регистрироваться, он продолжает запись как незарегистрированный посетитель.  |
| **Триггер** | Клиент выбирает вход или регистрацию во время записи  |

| **Выходные условия** | Минимальные гарантии: Если клиент не вошел и не зарегистрировался, он может продолжить запись как незарегистрированный посетитель.  Гарантии успеха: Клиент вошел в учетную запись. Либо клиент создал новую учетную запись. |
| **Нормальное направление развития варианта использования** | Система подтверждает номер телефона и предлагает осуществить вход или регистрацию в системе Клиент выбирает зарегистрироваться Система запрашивает данные для создания учетной записи  Клиент вводит необходимые данные Система подтверждает корректность введения данных и сохраняет учетную запись Система возвращает клиента к бронированию слота |
| **Альтернативное направление развития варианта использования** | 1.А.Система подтверждает номер телефона и предлагает осуществить вход или регистрацию в системе 1\. Клиент выбирает вход 2\.
